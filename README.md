## рҹҸ—пёҸ What is Site Stat?

> **Site Stat** is a purpose-built web application for civil engineering operations вҖ” a centralized command center that replaces slow, error-prone manual data entry with a fast, secure, and intelligent dashboard. From tracking daily labor across multiple shifts to generating site-wide analytics at a glance, Site Stat brings enterprise-grade tooling directly to the construction site.

No more clipboards. No more spreadsheets. No more guessing.

---

## вңЁ Core Features

### рҹ“Ҡ &nbsp;Real-Time Dashboard
A live, high-level overview of your entire operation вҖ” active workforce counts, daily site statistics, and critical KPIs вҖ” all updating in real time without a single page refresh.

### рҹ‘· &nbsp;Labor Tracking System
Log, manage, and archive daily labor data across projects and shifts with precision. Built-in historical records mean your workforce data is always audit-ready and instantly accessible.

### рҹ”җ &nbsp;Secure Backend Architecture
Powered by **Supabase** with PostgreSQL under the hood. Row Level Security (RLS) ensures your civil engineering data stays strictly confidential вҖ” accessible only to those who are authorized.

### рҹ“ұ &nbsp;Responsive Field UI
Designed for real conditions. Whether you're at a desktop in the office or on a phone at the job site, the clean, minimalist interface adapts seamlessly so nothing slows you down.

## рҹ’» Getting Started

### Prerequisites
- [Node.js](https://nodejs.org/) (v18 or higher recommended)
- A [Supabase](https://supabase.com/) project with your credentials ready

---

### Step 1 вҖ” Clone the Repository

```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
cd YOUR_REPO_NAME
```

### Step 2 вҖ” Install Dependencies

```bash
npm install
```

### Step 3 вҖ” Configure Environment Variables

Create a `.env` file in the root directory:

```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

> рҹ’Ў Find these values in your Supabase project under **Settings вҶ’ API**.

### Step 4 вҖ” Launch the Dev Server

```bash
npm run dev
```

The app will be live at **`http://localhost:5173`** рҹҡҖ

## рҹ”’ Security Model

Security is not an afterthought in Site Stat вҖ” it's baked into every layer:

- **Row Level Security (RLS):** Database-level policies ensure users can only access data they're explicitly authorized to see.
- **Supabase Auth:** Secure token-based authentication out of the box.
- **Environment Isolation:** All sensitive credentials are handled via environment variables вҖ” never hardcoded.
- **Edge Deployment:** No origin server to compromise; Vercel's infrastructure handles DDoS protection and TLS automatically.
